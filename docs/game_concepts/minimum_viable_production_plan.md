# Urchin Game Concept: Minimum Viable Production Plan

**Status:** Planning plan  
**Date:** 2026-04-24  
**Purpose:** Define the minimum viable production path for building the first slice of the game without losing the project’s identity through overreach.

---

## 1. Goal

The MVP is not “the game, but smaller.”

It is:

**the smallest playable thing that convincingly proves the game’s aesthetic, movement, and emotional identity.**

That means the MVP must prove:
- rolling feels good,
- the world looks distinct,
- the tone is working,
- resonance is special,
- the project is worth extending.

It does not need to prove:
- the whole story,
- all biomes,
- all vehicles,
- all puzzle families,
- the entire worldbuilding packet.

---

## 2. Recommended MVP Definition

The MVP should be a polished vertical slice made of:

1. a compact Moss Cathedral onboarding space,
2. a short Marble Dunes flow descent,
3. a small Sand Choir resonance reveal zone.

This is the narrowest slice that still contains:
- tenderness,
- movement joy,
- symbolic world response,
- a clear sense of future possibility.

---

## 3. MVP Must-Haves

### 3.1 Character

Need:
- final-enough readable urchin look,
- basic expressive motion states,
- working rolling,
- working pulse,
- basic resonance visual response.

Do not need yet:
- full late-game state suite,
- full carrier integration,
- advanced symbolic variants.

### 3.2 Environment

Need:
- one strong moss material family,
- one strong marble/mineral family,
- one strong sand/resonance family,
- atmospheric lighting,
- particles,
- a handful of iconic props.

Do not need yet:
- full biome library,
- every worldbuilding motif,
- all glyph categories.

### 3.3 Mechanics

Need:
- rolling
- pulse/compression
- one controlled flow run
- one resonance interaction mode
- one reveal puzzle

Do not need yet:
- skim
- glide
- vehicles
- advanced restraint puzzles
- synthesis-state chaining

### 3.4 Narrative

Need:
- awakening feeling
- one vulnerability beat
- one moment of joy
- one “the world is listening” reveal

Do not need yet:
- full beat sheet implementation
- revelation-state system
- full presence roster

---

## 4. MVP Build Pillars

The MVP should be built around these pillars:

### Pillar 1: Feel

If rolling and pulse are not satisfying, the rest does not matter.

### Pillar 2: Beauty

If the slice does not already feel visually distinct, the project loses its strongest advantage.

### Pillar 3: Response

The world must answer the player in a memorable way.

### Pillar 4: Tone

The slice must already feel sincere, quiet, and transportive.

---

## 5. Production Breakdown

## Phase A: Movement Core

Deliverables:
- controllable rolling
- pulse/compression
- camera behavior
- surface tuning for gentle and flow spaces

Success test:
- simply moving is enjoyable for several minutes

## Phase B: Visual Core

Deliverables:
- urchin material/look
- moss biome material kit
- marble biome material kit
- sand biome material kit
- atmosphere and particles

Success test:
- still screenshots already feel authored and distinct

## Phase C: Slice Geometry

Deliverables:
- awakening basin
- shelter corridor / root zone
- descent route
- resonance chamber
- final reveal overlook

Success test:
- the slice has a clear emotional arc without extra systems

## Phase D: Resonance Core

Deliverables:
- resonance pulse interaction
- visible ripple feedback
- buried path or structure reveal
- one glyph-guided puzzle

Success test:
- first use of resonance feels magical and readable

## Phase E: Polish Layer

Deliverables:
- audio tone pass
- FX pass
- scenic framing pass
- symbolic hint pass
- final composition at ending beat

Success test:
- the slice feels like a real game identity, not a prototype of systems only

---

## 6. Scope Reduction Rules

If production pressure rises, cut in this order:

1. optional side routes
2. symbolic presence glimpse
3. secondary prop variety
4. advanced resonance variants
5. extra chamber complexity

Do **not** cut first:
- rolling feel
- pulse feel
- atmosphere
- one beautiful descent
- one strong resonance reveal

Those are the identity.

---

## 7. MVP Asset Strategy

Keep the slice tractable by limiting asset families.

### Character

- one urchin hero asset
- small set of state variants through shader/animation, not duplicate models

### Environment

- 3 material families
- modular sculptural terrain pieces
- compact but reusable prop sets
- heavy use of atmosphere and particles

### FX

- pulse burst
- dust trail
- pollen drift
- sand ripple
- reveal glow

That is enough for a convincing slice.

---

## 8. MVP Design Risks

### Risk 1: Too much “game” before enough “feel”

If the slice prioritizes logic over motion pleasure, it will misrepresent the project.

### Risk 2: Too much atmosphere, not enough interaction

If the slice becomes only scenic wandering, it will under-prove the core systems.

### Risk 3: Visual ambition outruns implementation reality

If the style depends on realism or huge asset counts, production will stall.

### Risk 4: Too many mechanics

If skim, launch, or carriers creep into the MVP, the core identity will blur.

### Risk 5: Resonance too abstract

If the first resonance interaction is not tactile and readable, the strongest differentiator may fail.

---

## 9. MVP Success Criteria

The MVP is successful if a player or reviewer can say:

- “I immediately understood the tone.”
- “Rolling through this world feels beautiful.”
- “The resonance reveal was memorable.”
- “I want to continue into the next biome.”

It is not required that they say:
- “This already feels content-complete.”
- “I understand every system.”
- “It has the whole story.”

---

## 10. Recommended Next Production Docs

After MVP planning, the most useful follow-ups would be:

1. a slice task breakdown by system and art area,
2. a content blockout spec for the playable route,
3. a resonance implementation behavior sheet,
4. a style-locked image set for the first three slice areas.

---

## 11. Final Recommendation

The minimum viable production path is:

**Build the smallest beautiful thing that proves the body, the world, and the first miracle.**

For this game, that means:
- a tiny pilgrim body,
- a sculptural world,
- a joyful descent,
- and a moment when the sand answers back.
